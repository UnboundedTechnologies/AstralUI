Documentation

AstralUI (@astralui/core) is a self-contained, dependency-free React design system. Import the stylesheet once, wrap your app, and use the components. Everything is theme-aware (light/dark + per-brand) through --astral-* CSS variables and light-dark().

Install

pnpm add @astralui/core
# peer deps you likely already have:
pnpm add react react-dom @tabler/icons-react

Setup

Import the stylesheet once (entry file), then wrap your app:

import '@astralui/core/styles.css';
import { ColorSchemeProvider, AstralThemeProvider, AstralToaster } from '@astralui/core';

function Root() {
  return (
    <ColorSchemeProvider defaultScheme="dark">
      <AstralThemeProvider>
        <AstralToaster />
        <App />
      </AstralThemeProvider>
    </ColorSchemeProvider>
  );
}

Color scheme

ColorSchemeProvider persists the scheme and sets data-astral-scheme + color-scheme on <html>. Toggle it anywhere with useColorScheme().

const { colorScheme, toggle, setColorScheme } = useColorScheme();

Per-brand theming

Pass any subset of brand colors; AstralUI builds a 10-shade palette via generateColors() and overrides the tokens per scheme. useThemePreview().setPreview(...) applies a live, unsaved preview.

<AstralThemeProvider colors={{
  primary_color: '#7950f2',
  background_color: '#141417', card_color: '#1c1c20', font_color: '#f4f4f5',
  light_primary_color: '#0f766e', light_background_color: '#f5f7fa' /* ...light_* */
}}>

Design tokens

Use the variables directly in your own CSS. Core tokens: --astral-color-text, --astral-color-body, --astral-color-dimmed, --astral-color-default-border, the --astral-color-{hue}-{0..9} palette, --astral-primary-color-filled, and --astral-font-family.

.thing { background: var(--astral-color-body); color: var(--astral-color-text);
         border: 1px solid var(--astral-color-default-border); }

Modal & Drawer

Portalled, centered modal / right-side drawer with escape + click-outside. Props: opened, onClose, title, width?, brand?, zIndex?, children.

<AstralModal opened={open} onClose={close} title="Edit profile" width={460}>
  ...
</AstralModal>

<AstralDrawer opened={open} onClose={close} title="Filters" width={440}>...</AstralDrawer>

ConfirmModal

Confirmation / alert dialog built on AstralModal. Set danger for a red confirm button and loading while an async action runs.

<ConfirmModal opened={open} onClose={close} onConfirm={remove}
  title="Delete campaign?" message="This cannot be undone."
  confirmLabel="Delete" danger loading={busy} />

Select

Single-select combobox (searchable / clearable / creatable), portalled so it floats above modals.

<AstralSelect value={v} onChange={setV} searchable clearable
  options={[{ value: 'us', label: 'United States' }]} />

Menu

Anchored dropdown from an items array. Items support icon, danger, disabled, and dividers.

<AstralMenu items={[
  { label: 'Edit', onClick: edit },
  { label: 'Delete', danger: true, onClick: remove },
]} />

PinInput

Code / OTP input with auto-advance, backspace, paste, and arrow keys.

<AstralPinInput length={6} value={code} onChange={setCode} />

Notifications

Mount <AstralToaster /> once at the root, then call notifications anywhere. A loading toast stays until you update it.

notifications.show({ message: 'Saved', color: 'green' });
const id = notifications.show({ message: 'Importing...', loading: true, autoClose: false });
notifications.update({ id, message: 'Done', color: 'green', loading: false, autoClose: 2000 });

DateInput

Freeze-proof native date / datetime-local input. Commits a parsed Date only on a complete value. Helpers: dateToInputStr, dateTimeToInputStr.

<DateInput type="date" value={date} onChange={setDate} />

Spinner

<Spinner size={20} />   // or the .au-spinner class

Charts

Dependency-free, prop-driven, theme-aware. Colors accept a hex, a CSS var(), or a palette token like "violet.6".

// KPI sparkline + period delta
<Sparkline points={[3,5,4,8,7,11]} color="violet.6" />
<Delta current={420} previous={377} />

// interactive, linked-hover charts
<Donut centerValue="1,284" centerLabel="sent" data={[
  { name: 'Accepted', value: 412, color: 'green.6' },
  { name: 'Pending', value: 872, color: 'gray.6' },
]} />
<FunnelBars data={[{ stage: 'Sent', value: 1284, color: 'violet.6' }]} />
<StackedBars series={[{ label: 'A', color: 'violet.6' }]} rows={[{ label: 'Mon', values: [12] }]} />

Auth shell

AuthShell + AuthCard give a centered sign-in / sign-up / onboarding layout on a cosmic or wash backdrop. Build the form with the au-auth-* classes (au-auth-field, au-auth-input, au-auth-btn, au-auth-note, au-auth-steps).

<AuthShell backdrop="space" topRight={<LangSwitcher />}>
  <AuthCard banner={<Logo />}>
    <h1 className="au-auth-title">Welcome back</h1>
    <form className="au-auth-form">
      <div className="au-auth-field">
        <label>Email</label>
        <div className="au-auth-input"><input /></div>
      </div>
      <button className="au-auth-btn primary block">Sign in</button>
    </form>
  </AuthCard>
</AuthShell>

Backgrounds

Full-bleed decorative backdrops. Wrap content as children; add fixed to pin to the viewport.

<SpaceBackground fixed />        // animated starfield + cosmic glow
<GridBackground>...</GridBackground> // brand-tinted, edge-masked technical grid

Or use the classes directly: .astral-space-bg + .astral-stars, .astral-grid-bg.

Utility classes

The package also ships the full primitive layer as classes you can use without a component: buttons (.au-btn, .au-btn.primary/.danger/.ok/.green), inputs (.au-input, .au-textarea, .au-field-label), .au-status pills, .au-tag, .au-badge-ok, .au-alert, .au-table, .au-toggle, .au-card, .au-panel, .au-empty, notification tiles (.au-ntf), and more.
